👥 Role Definitions & Access Levels

🔐 SUPER_ADMIN

Scope: Platform-wide (All schools)

Manages all schools, system settings, features, billing, and user access across the entire platform.

🏫 SCHOOL_ADMIN

Scope: Single school (Tenant)

Manages school operations, staff, students, academics, fees, and assigns roles/features to users within their school.

🤝 ASSISTANT

Scope: Operational support

Front-desk operations: student records, attendance corrections, fee collection, facility requests, announcements.

👨‍🏫 TEACHER

Scope: Assigned classes

Manage attendance, assign homework, create exams, publish results, communicate with parents and staff.

👨‍👩‍👧 PARENT

Scope: All children in same school

Login: Parent account (no student logins). Switch between children to view ALL data: attendance, homework, results, timetable, fees, transport. Can see multiple children's complete information.

🔗 Master Relationships

schools → Root master table. All data is scoped by school_id for multi-tenancy.
users → Connects to schools via school_id. Roles via user_roles table (many-to-many).
students → FK to schools & classes. Each student belongs to one school (tenant isolation via school_id).
classes → FK to schools & academic_years. Represents a specific grade/section in a school.
invoices → FK to students & fee_structures. Every invoice links back to a student (which links to school).
parent_id (in students) → FK to users(id). Connects student to parent account. Parents LOGIN via user account and see ALL their children's data. NO separate student logins.
